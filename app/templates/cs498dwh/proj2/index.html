Random and Overlapping methods of Texture image generation

Examples

For the randomly sampled texture, we randomly select a patch of specified size patchsize which leads to crude image. This can be improved by finding the next patch using a previous patch using a certain part of the patch as overlap, and then finding the Sum of squared difference between the patch and texture to find the best possible patch. The filter operation is used to calculate the ssd. This does not always turn out well, as you can see in the images below, but is still better than randomly sampling.
